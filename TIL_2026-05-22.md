# TIL - 2026-05-22

## 오늘 실습한 내용

오늘은 JavaScript의 기본 문법부터 스코프와 객체/배열 조작까지 순서대로 실습했다.

### 1. 반복문
- `while`, `for`의 기본 구조를 복습했다.
- 배열을 순회할 때는 `arr.length`를 기준으로 `0`부터 `length - 1`까지 접근해야 한다는 점을 확인했다.
- `continue`는 특정 조건에서 건너뛰기, `break`는 반복 종료에 사용했다.
- 역방향 순회와 중첩 `for`문도 작성해 보면서 표 형태 데이터 탐색 방식도 같이 익혔다.

### 2. `for...in` / `for...of`
- `for...in`은 키를 순회할 때 사용하고, `for...of`는 값을 순회할 때 사용한다.
- 배열에서는 둘 다 쓸 수 있지만, 일반적으로는 `for...of`가 값 순회에 더 적합하다.
- 객체는 기본적으로 `for...of`에 바로 넣을 수 없어서 `Object.keys()`, `Object.values()`, `Object.entries()` 같은 도구가 필요하다.

### 3. 함수와 스코프
- 함수 안에서 선언한 변수는 함수 밖에서 접근할 수 없다는 점을 확인했다.
- 매개변수와 지역변수는 함수 블록 안에서만 유효하다.
- `return`은 함수의 실행을 끝내고 값을 돌려주는 역할을 한다.
- `switch(true)` 형태로 조건 분기를 처리하는 방식도 실습했다.

### 4. 매개변수와 인자
- 함수는 인자를 적게 넣어도 동작하고, 넘치는 인자는 무시된다는 점을 확인했다.
- 기본값 매개변수(`a = 120`)를 사용해 초기값을 지정할 수 있었다.
- `arguments` 객체로 실제 전달된 인자 목록을 확인했다.
- rest parameter(`...b`)로 나머지 인자를 배열로 받을 수 있었다.

### 5. 함수 표현식, 호이스팅, 화살표 함수
- 함수 선언문과 함수 표현식의 차이를 확인했다.
- 호이스팅 때문에 `function` 선언은 선언 전에 호출 가능하지만, `let`/`const`와 함수 표현식은 그렇지 않다.
- `var`는 함수 스코프를 가지며, `let`/`const`는 블록 스코프를 가진다.
- 화살표 함수는 간단한 표현에 적합하고, `this` 동작이 일반 함수와 다를 수 있다는 점을 봤다.

### 6. 클로저
- 렉시컬 스코프 개념을 통해 함수가 선언된 위치를 기준으로 스코프가 결정된다는 점을 정리했다.
- 반환된 내부 함수가 외부 변수 `c`를 계속 기억하는 클로저를 구현했다.
- 상태를 외부에 노출하지 않고도 유지할 수 있어서 버튼 상태 관리나 카운터 구현에 유용하다는 점을 이해했다.

### 7. 객체
- 객체의 프로퍼티 추가, 수정, 삭제를 실습했다.
- 메서드를 객체 내부에 정의하고 호출하는 방식을 확인했다.
- `Object.keys()`, `Object.values()`, `Object.entries()`로 객체를 순회하는 방법을 익혔다.
- 구조 분해 할당으로 필요한 값만 꺼내고, 나머지는 `...rest`로 받을 수 있었다.

### 8. 배열
- `push`, `pop`, `shift`, `unshift`, `splice`로 배열을 다루는 방법을 실습했다.
- 배열도 내부적으로는 키 기반 구조라서 `Object.keys()`와 `Object.entries()`로 확인할 수 있었다.
- `splice()`는 중간 구간을 제거하거나 추출할 때 유용했다.

## 오늘 배운 핵심

- 반복문은 단순히 돌리는 것보다 "언제 끝나는지"를 명확히 잡는 게 중요하다.
- `for...in`은 키, `for...of`는 값이라는 차이를 분명히 구분해야 한다.
- 함수는 값처럼 다룰 수 있고, 선언 방식에 따라 호출 가능 시점이 달라진다.
- 스코프와 호이스팅을 이해해야 `var`, `let`, `const`의 차이를 헷갈리지 않는다.
- 클로저는 상태를 유지하는 데 강력하다.
- 객체와 배열은 구조 분해, 전개 연산자, 내장 메서드를 함께 써야 실전에서 편하다.

## 아쉬웠던 점

- `var`와 `let`의 차이를 코드 실행 순서 관점에서 더 많이 연습할 필요가 있다.
- `for...in`과 `for...of`를 객체/배열별로 자연스럽게 구분해서 쓰는 연습이 더 필요하다.
- 클로저는 개념은 이해했지만, 실제 활용 예제를 더 많이 만들어 봐야 한다.
